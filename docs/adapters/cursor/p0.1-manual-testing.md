# P0.1a — Snapshot Restore (Manual Live Checklist)

Scope: **settings.set** and **fs.write** (overwrite snapshot only). No terminal, agent, or workspace.open restore.

## Prerequisites

- Bridge API running (`npm run serve` or `npm run dev:api`)
- Bridge Cursor extension installed and IDE Control host active (port 3848)
- Client ID with `settings` and `fs-write` permissions (`bridge-default` in dev)

## 1. Version / registry

```http
GET /api/v1/cursor/version
```

Expect: `snapshotRestoreAvailable: true`, `rollbackAvailable: true`, `registryVersion: poc-v1.2.0`

```http
GET /api/v1/cursor/registry
```

Expect: `cursor.ide.settings.set` and `cursor.ide.fs.write` have `rollbackAvailable: true`; `workspace.open` remains `false`.

## 2. Restore settings.set

1. Change a workspace setting (creates snapshot):

```http
POST /api/v1/cursor/ide/settings/set
Content-Type: application/json

{
  "section": "editor",
  "key": "tabSize",
  "value": 8,
  "confirmed": true
}
```

Note `snapshotId` in the response.

2. Restore:

```http
POST /api/v1/cursor/snapshots/{snapshotId}/restore
Content-Type: application/json

{ "confirmed": true }
```

Expect: `200`, `ok: true`. Editor tab size returns to the previous value.

3. Retry restore (idempotent): same request should succeed again; snapshot file is **kept**.

## 3. Restore fs.write (overwrite only)

1. Write to an existing file with overwrite:

```http
POST /api/v1/cursor/ide/fs/write
Content-Type: application/json

{
  "path": "<allowlisted-path>",
  "content": "new content",
  "overwriteExisting": true,
  "confirmed": true
}
```

Expect `snapshotId` only when the file existed before write.

2. Restore via `POST /api/v1/cursor/snapshots/{snapshotId}/restore` with `{ "confirmed": true }`.

Expect: file content reverts to pre-write content. Requires extension online (no FS fallback for fs.write).

## 4. Negative cases

| Case | Request | Expect |
|------|---------|--------|
| Unknown snapshot | `POST .../snapshots/bad-id/restore` + `confirmed: true` | 404 `SNAPSHOT_NOT_FOUND` |
| No confirmation | restore without `confirmed: true` | 428 `CONFIRMATION_REQUIRED` |
| Workspace snapshot | restore snapshot from `workspace.open` (if created manually) | 400 `SNAPSHOT_UNSUPPORTED` |
| fs.write, extension offline | restore fs.write snapshot with extension stopped | 503 `EXTENSION_UNREACHABLE` |

## 5. Audit

```http
GET /api/v1/cursor/audit?actionId=cursor.snapshots.restore
```

Expect entries for successful/failed restores with `actionId: cursor.snapshots.restore` (meta action, not in the 10-action registry).

## 6. Settings FS fallback (extension offline)

1. Stop extension / disconnect IDE Control host.
2. `POST .../ide/settings/set` with filesystem fallback (if path allowlist permits).
3. `POST .../snapshots/{snapshotId}/restore` with `confirmed: true`.

Expect: restore succeeds via **workspace** `.vscode/settings.json` merge without extension.

**Restriction:** FS fallback applies only when the snapshot `target` is `workspace` (or omitted, defaulting to workspace). Snapshots with `target: "global"` or `target: "workspaceFolder"` require the Extension; restore returns `EXTENSION_UNREACHABLE` if the host is offline.
